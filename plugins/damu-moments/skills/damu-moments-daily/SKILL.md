# 达目社朋友圈日更

## 开始前

1. 完整读取 [writing-rules.md](references/writing-rules.md)。需要调用知识接口或处理异常时，再读取 [api-contract.md](references/api-contract.md)。
2. 使用 `node scripts/damu-moments.mjs status` 检查知识索引。索引不可用时说明原因，不虚构达目社课程内容。
3. 首次使用时询问并保存用户愿意提供的身份、主要受众、日常口吻、业务方向和禁用表达。不要索取身份证、手机号、住址等无关隐私。
4. 只把用户明确确认发布过的文案写入本地历史；草稿和候选稿不算已发布。

## 每日日更流程

1. 获取当天真实锚点：用户今天的一件小事、一个观察、一段对话或一个真实困惑。用户没有可提供的经历时，改写为中性知识分享，不伪造第一人称故事。
2. 从锚点提取 2—4 个检索词，运行：

   ```bash
   node scripts/damu-moments.mjs search --query "<检索词>" --limit 8
   ```

3. 只依据返回的 `title` 与 `excerpt` 写作。至少选择一个来源，记录其 `id`、标题和索引 `revision`；不得把来源摘要扩写成不存在的数据、案例或效果。
4. 设计三个明显不同的候选角度：真实日常、认知观点、互动提问。避免同义改写成三条看似不同的文案。
5. 对每条候选稿运行：

   ```bash
   node scripts/damu-moments.mjs check --text "<候选稿>" --source-ids "<来源ID>" --angle "<角度>"
   ```

6. 命中以下任一条件时重写：相同来源在 7 天内无新角度、相同角度在 30 天内使用过、文本相似度不低于 `0.72`、开头与最近内容高度相似。
7. 按 [daily-output-template.md](assets/daily-output-template.md) 交付三版文案，并附来源依据、去重结论和配图建议。来源依据不属于公开朋友圈正文。
8. 用户明确回复“用了第 N 版”“已经发了”后，再运行：

   ```bash
   node scripts/damu-moments.mjs record --confirmed --text "<已发布正文>" --source-ids "<来源ID>" --angle "<角度>"
   ```

## 修改与检查

- 用户要求“换个角度”时保留事实来源，重新选择叙事场景、开头手法和结尾动作，不只替换近义词。
- 用户提供现成文案时，先做事实、红线和历史重复检查，再给出最小必要修改。
- 用户要求产品种草时，使用可核验的产品资料；没有产品卡或官方依据时只写选择方法，不写具体功效。
- 用户要求自动发布朋友圈时，只交付文案和配图建议。除非另有经过授权且可确认的发布工具，否则不得声称已经发布。

## 失败边界

- 知识接口失败且没有 24 小时内缓存时，暂停基于知识库的写作并说明故障。
- 返回来源不足时缩小题目，不用常识冒充达目社知识库内容。
- 不输出接口令牌、飞书凭证、本地历史文件原文或其他用户数据。
